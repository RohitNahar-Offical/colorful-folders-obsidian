# 🎨 UI & Styling System

This document outlines the design language, CSS architecture, and animation tokens used in Colorful Folders.

## 1. CSS Architecture

We use a "Layered CSS" approach. Styles are injected into three distinct areas:

1.  **Global Base Styles**: (Native Obsidian styles + `styles.css`) - Handles core layout and variables.
2.  **Dynamic Engine Styles**: (Generated by `StyleGenerator`) - Handles path-specific coloring and icons.
3.  **UI Component Styles**: (Generated by `main.ts` into `uiStyleTag`) - Handles custom components like Dividers and Modals.

---

## 2. Design Tokens (CSS Variables)

We define several custom variables to ensure consistency across light and dark modes:

| Variable | Description |
| :--- | :--- |
| `--cf-divider-color` | The base color for divider lines. |
| `--cf-popover-bg` | Background for the Markdown hover popover. |
| `--cf-active-glow` | Color for the path connecting line. |
| `--cf-icon-scale` | Multiplier for all folder/file icons. |

---

## 3. Injected UI Components

### Interactive Dividers (`.cf-interactive-divider`)
*   **Structure**: 
    *   `.cf-divider-line-left` / `.cf-divider-line-right`
    *   `.cf-divider-chip` (The rounded "pill" container)
    *   `.cf-divider-icon` (Optional Lucide/Custom icon)
*   **Behavior**: Transitions from `opacity: 0.7` to `1.0` on hover. The chip scales slightly on hover.

### Premium Popovers (`.cf-premium-popover`)
*   **Design**: Uses `backdrop-filter: blur(12px)` and a subtle border gradient.
*   **Positioning**: Calculated dynamically to prevent overflow off the screen. Uses `z-index: var(--layer-popover)`.

## 4. Animations

We provide several "Active Path" animation styles:

### Shimmer (`@keyframes cf-shimmer`)
A linear gradient that slides across the active file title.
```css
@keyframes cf-shimmer {
    0% { background-position: -200% 0; }
    100% { background-position: 200% 0; }
}
```

### Pulse (`@keyframes cf-pulse`)
A soft breathing effect applied to the path-connecting line.

## 5. Icon Rendering (The Mask Trick)

To support any color for any icon (even custom ones), we use `mask-image`.

**The Problem**: SVGs have their own colors hardcoded inside.
**The Solution**:
```css
.my-icon {
    background-color: var(--target-color); /* We set the color here */
    mask-image: url('data:image/svg+xml;utf8,...'); /* The icon is a hole */
    mask-size: contain;
    mask-repeat: no-repeat;
}
```
*Note: We use `-webkit-mask-image` for compatibility with Obsidian's Electron environment.*

---

## 6. Glassmorphism Logic

When `glassmorphism: true` is enabled, we apply:
```css
background-color: rgba(..., 0.4) !important;
backdrop-filter: blur(10px) saturate(180%);
```
*Tip: This works best with semi-translucent community themes.*
