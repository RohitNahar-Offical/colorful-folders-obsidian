# 🎨 UI and Styling System

This document outlines the design language, CSS architecture, and animation tokens used in **Colorful Folders**.

---

## 1. CSS Architecture

We use a **Layered CSS** approach. Styles are injected into three distinct areas:

1.  🌐 **Global Base Styles**: (Native Obsidian + `styles.css`) - Core layout and variables.
2.  🚀 **Dynamic Engine Styles**: (Generated by `StyleGenerator`) - Path-specific coloring and icons.
3.  🛠️ **UI Component Styles**: (Generated by `main.ts` into `uiStyleTag`) - Custom widgets like Dividers and Modals.

---

## 2. Specificity and the `!important` Rule

> [!NOTE]
> Because Obsidian's native theme uses highly specific selectors, we use `!important` judiciously.

- **Background Tints**: Ensures colors are visible over active-state theme colors.
- **Icon Masks**: Guarantees custom icons override theme-provided ones.
- **Stealth Mode**: Forces `display: none` regardless of other layout rules.

---

## 3. Design Tokens (CSS Variables)

| Variable | Description | Default |
| :--- | :--- | :--- |
| `--cf-divider-color` | Base color for divider lines | `var(--background-modifier-border)` |
| `--cf-popover-bg` | Background for Markdown hover popover | `var(--background-secondary)` |
| `--cf-active-glow` | Color for the path connecting line | `var(--interactive-accent)` |
| `--cf-meta-weight` | Weight for tags, extensions, and counters | **900** |

---

## 4. Injected UI Components

### 📏 Interactive Dividers (`.cf-interactive-divider`)
- **Structure**: Rounded "pill" chip between two lines.
- **Connectivity**: Supports gaps as low as **-10px** for seamless intersection.
- **Behavior**: Subtle scaling and opacity transition on hover.

### 💨 Premium Popovers (`.cf-premium-popover`)
- **Design**: Uses `backdrop-filter: blur(12px)` and a subtle border gradient.
- **Positioning**: Calculated dynamically to prevent overflow.

---

## 5. Premium Static Visuals
 
We have decommissioned dynamic animations in favor of high-performance static visual standards that ensure zero GPU overhead.
 
### 🌟 Luminous Selection
The currently active file is highlighted with a multi-layered static effect:
- **Bevel Sheen**: A subtle `linear-gradient` overlay mimicking a 3D glass edge.
- **Luminous Glow**: A path-synced `box-shadow` that creates a "light source" effect without looping keyframes.
 
### 📏 Radiant Path
The connecting vertical trail is a strictly standardized `2px` solid line.
- **Visual Stability**: By using a static thickness and color, we eliminate the "jitter" and "thinning" artifacts caused by CSS animation engines during scrolling.

---

## 6. Icon Rendering (The Mask Trick)

To support any color for any icon, we use CSS masking.

> [!TIP]
> This allows us to render hundreds of icons with zero DOM overhead.

```css
.my-icon {
    background-color: var(--target-color);
    -webkit-mask-image: url('data:image/svg+xml;...');
    mask-size: contain;
    mask-repeat: no-repeat;
}
```

---

## 7. Glassmorphism Logic

When `glassmorphism: true` is enabled, we apply:
```css
background-color: rgba(..., 0.4) !important;
backdrop-filter: blur(10px) saturate(180%);
```

---

## 8. UI Copy Policy (Sentence Case)

To maintain a native feel, all UI text must follow **Sentence Case**:
- **Correct**: `Enable automatic icons` ✅
- **Incorrect**: `Enable Automatic Icons` ❌

---

> [!IMPORTANT]
> All new UI components must be tested in both **Light** and **Dark** modes to ensure contrast accessibility via the `utils.ts` contrast logic.
